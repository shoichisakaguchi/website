# YAML Formatting

- Use 2-space indentation; never use tabs.
- Quote strings that contain `:` or start with special YAML characters (`-`, `{`, `}`, `[`, `]`, `#`, `&`, `*`, `!`, `|`, `>`, `%`, `@`, `,`).
- Use double quotes for quoted strings in frontmatter for consistency.
- Prefer block scalars (`|-` or `>-`) for multi-line text.
- Keep empty objects/arrays explicit (`{}` or `[]`) only when required by schema or intentional.
